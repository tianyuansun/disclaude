## 1. 创建飞书应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 点击「开发者后台」进入应用管理
3. 点击「创建企业自建应用」
4. 填写应用信息：
   - **应用名称**: 建议使用 `Disclaude` 或自定义名称
   - **应用描述**: AI Agent 助手机器人
   - **应用图标**: 上传自定义图标（可选）

> **提示**: 如果是 Lark（海外版），请访问 [Lark Developer](https://open.larksuite.com/)

---

## 2. 获取应用凭证

创建应用后，在「凭证与基础信息」页面获取：

| 凭证 | 说明 | 用途 |
|------|------|------|
| **App ID** | 应用唯一标识 | 配置文件中的 `feishu.appId` |
| **App Secret** | 应用密钥 | 配置文件中的 `feishu.appSecret` |

> ⚠️ **安全提示**: App Secret 非常重要，请妥善保管，不要提交到代码仓库！

---

## 3. 启用机器人能力

1. 在应用管理页面，点击左侧菜单「应用功能」→「机器人」
2. 开启「启用机器人」开关
3. 配置机器人信息：
   - **机器人名称**: 与应用名称一致
   - **机器人描述**: 描述机器人功能
   - **机器人头像**: 上传图标（可选）

---

## 4. 配置权限

在「权限管理」→「权限配置」页面，申请以下权限：

### 4.1 消息相关权限

| 权限 Scope | 权限名称 | 说明 | 必需 |
|-----------|---------|------|------|
| `im:message` | 获取与发送消息 | 发送、回复、编辑消息 | ✅ 必需 |
| `im:message:send_as_bot` | 以应用身份发消息 | 以机器人身份发送消息 | ✅ 必需 |
| `im:message:readonly` | 读取消息 | 获取消息内容 | ✅ 必需 |
| `im:resource` | 获取消息资源 | 下载消息中的图片、文件 | ✅ 必需 |
| `im:reaction` | 获取与更新表情反应 | 添加消息表情 | 可选 |

### 4.2 文件相关权限

| 权限 Scope | 权限名称 | 说明 | 必需 |
|-----------|---------|------|------|
| `im:image` | 获取与上传图片或文件 | 上传/获取图片 | ✅ 必需 |
| `im:file` | 获取与上传文件 | 上传文件到消息 | ✅ 必需 |
| `drive:file:readonly` | 查看和下载云空间中的文件 | 下载云盘文件 | 可选 |

### 4.3 群组相关权限

| 权限 Scope | 权限名称 | 说明 | 必需 |
|-----------|---------|------|------|
| `im:chat` | 获取与更新群组信息 | 创建、删除、获取群组 | ✅ 必需 |
| `im:chat:member` | 获取与更新群组成员 | 管理群组成员 | ✅ 必需 |
| `im:chat:readonly` | 获取群组信息 | 读取群组列表 | ✅ 必需 |

### 4.4 权限申请步骤

1. 在「权限配置」页面，搜索权限名称或 Scope
2. 点击「添加权限」
3. 对于敏感权限，需要提交申请理由：
   ```
   本应用为 AI 编程助手，需要消息收发、文件处理、群组管理等功能以提供完整的对话式开发体验。
   ```
4. 等待管理员审批（如有）

## 5. 配置事件订阅

Disclaude 使用 WebSocket 长连接模式接收消息，无需公网服务器。

### 5.1 配置步骤

1. 在应用管理页面，点击「事件与回调」→「事件订阅」
2. **订阅方式**: 选择「使用长连接接收事件」
3. **添加事件**: 点击「添加事件」，搜索并添加：
   - `im.message.receive_v1` - 接收消息

### 5.2 事件说明

| 事件 | 说明 | 用途 |
|------|------|------|
| `im.message.receive_v1` | 接收群聊或私聊消息 | 接收用户发送的消息 |

## 6. 发布应用

### 6.1 创建版本

1. 在应用管理页面，点击「版本管理与发布」
2. 点击「创建版本」
3. 填写版本信息：
   - **版本号**: 如 `1.0.0`
   - **更新说明**: 初始版本发布
4. 点击「保存」

### 6.2 申请发布

1. 点击「申请线上发布」
2. 等待管理员审批
3. 审批通过后，应用即可在飞书中使用

### 6.3 添加到群聊

1. 在飞书群聊中，点击群设置（右上角 ⋯）
2. 点击「群机器人」→「添加机器人」
3. 选择你创建的机器人

---

## 7. 配置 Disclaude

### 7.1 创建配置文件

```bash
# 复制示例配置
cp disclaude.config.example.yaml disclaude.config.yaml
```

### 7.2 填写飞书凭证

编辑 `disclaude.config.yaml`：

```yaml
feishu:
  # 从飞书开放平台获取
  appId: "cli_xxxxxxxxxxxxxxxx"      # 替换为你的 App ID
  appSecret: "xxxxxxxxxxxxxxxxxxxx"  # 替换为你的 App Secret

  # 可选：测试用的聊天 ID
  cliChatId: ""
```

### 7.3 启动服务

```bash
# 构建并启动
npm run build && npm start

# 或使用 PM2（生产环境推荐）
npm run pm2:start
```

### 7.4 验证配置

1. 在飞书中找到你的机器人
2. 发送消息 `你好` 或 `/help`
3. 如果收到回复，说明配置成功！

## 8. 常见问题

### Q1: 消息发送失败，提示权限不足

**原因**: 权限未正确配置或未生效

**解决方案**:
1. 检查「权限配置」页面，确认所有必需权限已添加
2. 检查权限状态是否为「已生效」
3. 如果权限需要审批，等待管理员审批通过
4. 重新发布应用版本

### Q2: 收不到消息事件

**原因**: 事件订阅未正确配置

**解决方案**:
1. 检查「事件订阅」页面，确认订阅方式为「长连接」
2. 确认 `im.message.receive_v1` 事件已添加
3. 检查 Disclaude 服务是否正常运行
4. 查看服务日志是否有错误信息

### Q3: 机器人没有出现在群聊中

**原因**: 应用未发布或未添加到群

**解决方案**:
1. 确认应用已发布到线上版本
2. 在群设置中手动添加机器人
3. 检查机器人是否被禁用

### Q4: 上传图片/文件失败

**原因**: 文件相关权限未配置

**解决方案**:
1. 添加 `im:image` 和 `im:file` 权限
2. 如果需要下载云盘文件，添加 `drive:file:readonly` 权限
3. 重新发布应用版本

### Q5: 无法创建群组或管理成员

**原因**: 群组相关权限未配置

**解决方案**:
1. 添加 `im:chat` 和 `im:chat:member` 权限
2. 这些权限可能需要管理员审批
3. 提供合理的申请理由

### Q6: WebSocket 连接失败

**原因**: 网络问题或凭证错误

**解决方案**:
1. 检查网络连接是否正常
2. 确认 App ID 和 App Secret 填写正确
3. 检查服务日志中的错误信息
4. 确认应用状态为「已启用」

### Q7: 海外版 Lark 配置

**差异**: Lark 和飞书的域名不同

**配置**:
- 开放平台地址: `https://open.larksuite.com/`
- API 域名会自动适配，无需额外配置
